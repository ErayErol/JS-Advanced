Какво е База данни?
=> Някаква колекция от данни, организирани и структурирани по начин по който да могат да се 
управляват лесно, да се осъществява бързо достъп до данни, да се търси, да се извлича информация, лесно да се променят и т.н.
=> Готова функционалност, когато ние създаваме проложения, които работят с таблици с колекции от някакви обекти и т.н.
и искаме тези данни да ги записваме персистентно на хард диска или на някакво друго място (отдаличен сървър, cloud и т.н.)
=> Съвременните база данни се управляват от Database Management System (DBMS)
DBMS => таблица, която върши цялата работа, дефинира ти таблици, извлича ти данни от таблици, променя ти таблици, изтрива и т.н.
Всички DBMS правят 4-те основни операции (CRUD)
В контекста на бек енд ние имаме API за CRUD операции
В контекста на база данни ние имаме SQL заявки или ORM система за CRUD операции

DB server
=> Client - Server
=> Client е нашето приложение например softuni judge
=> Server е базата данни например MS SQL server
=> Клиента се свързва с сървъра и DB Engine достъпва файловете
Данните се обработват, форматират, приготвят в някъв изходен вид и се връщат към клиента
=> DB server-ите задават структура на базата данни, те дефинират 
таблици или други обекти като колекции, в тези обекти има пропъртита или свойства или колони, релации, индексии и много други неща

Релационни база данни (RDBMS)
=> Те държат таблици и релации между тях, в таблиците има редове с данни

Нерелационни база данни (NO SQL)
=> Те съдържат колекции от документи или key-value, тоест обекти
