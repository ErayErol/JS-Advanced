Протокол в програмирането 
=> множество от правила за комуникацията между обекти, програми или други програмни единици. (уики)
=> са езици, с които си комуникират машините (наков)
=> както ние хората си комуникираме с думи и се разбираме, машините си комуникират една с друга с протоколи (аз)

HTTP 
=> е мрежов протокол, за пренос на информация в компютърни мрежи (уики)
=> интернет протокол използван за комуникация между уеб сървъра и уеб клиента, 
който взаимодейства с тях и показва крайният резултат на екрана (наков)
=> В HTTP протокола се използват понятия като клиент (обикновено това са Web-браузърите (или web навигаторите)
т.е. самите приложения, a не физическите хостове в мрежата) и сървър (това са уеб сървърите, т.е. самите приложения, а не хостовете).
=> Web client => HTTP Request => Web server => HTTP Response => Web client
Винаги комуникацията е такава както горе е описано
Никога не е възможно сървъра да изпрати нещо без клиента да е поискал

Клиент
=>Клиент в информатиката е хардуерен или софтуерен компонент на компютърна система, изпращащ заявки към сървър.
Програмата, която е клиент, си взаимодейства със сървъра посредством определен протокол. 
Тя може да поиска от сървъра някакви данни, да ги обработва непосредствено на сървъра, да стартира на сървъра нови процеси и т.н. 
Получените от сървъра данни може да се предоставят на потребителя или да се използват по друг начин в зависимост от предназначението на програмата. 
Програмата-клиент и програмата-сървър могат да работят както на един и същ компютър, така и на различни. 
Във втория случай за обмен на информация между тях се използва компютърна мрежа. 
Например при сърфиране в Интернет, браузърът играе ролята на клиент, който подава заявка по определен комуникационен протокол към сървъра, 
където са разположени търсените уеб страници.
=> Уеб клиент => уеб браузър, мобилно приложение

Уеб сървър 
=> Софтуер, който хваща заявките от уеб клиента и ги дава на бек-енда 
=> поисканият файл може да е статичен ресурс => HTML, PDF, JPG
=> може да е динамичен, който идва от базата данни
=> след като бекенд процеса приключи ги дава => на сървъра, който ги дава => на уеб клиента
=> Уеб сървър предоставя информация на клиент под формата на HTML документ. 
За пренасянето на информацията се използва протокола HTTP. 
Често уеб сървър се използва за предоставяне на информация и в други формати 
най-често това са изображения във форматите PNG, JPEG и GIF; XML документи и др.
=> Уеб сървърът по замисъл е направен да доставя уеб страници на уеб браузър. 
Цялата комуникация е възможна чрез обмен на HTTP requests, отправени от браузъра и 
HTTP responses от страна на сървъра, което осигурява комуникацията съгласно приет стандарт.

Frond-End
=> Web front-end е термин, описващ частта от уеб приложение или страница, която е видима за крайния потребител
Back-end  
=> се грижи за вътрешната инфраструктура на сайта, начин на функциониране на сайта, 
сигурност, управление на съдържанието, актуализации, база данни, сървър и др.

Frond end и Back end си комуникират през HTTP почти винаги

Уеб браузърът ползва URL, a HTTP ползва URI
